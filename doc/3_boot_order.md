# Реализовать раздел "Порядок загрузки" формы "Создание ВМ"

Раздел должен быть размещён в отдельной Card.
У раздела должен быть заголовок: "Порядок загрузки".

В этом разделе отображаются диски, ранее заданные в разделе "Диски HDD".
Для каждого диска показывается только его название (name).
Диски отображаются в вертикальном списке.
Должна быть возможность менять порядок дисков в списке с помощью drag-n-drop.

Меняя порядок дисков в этом разделе, пользователь указывает, какой будет порядок попыток загрузки для создаваемой виртуальной машины.

В состоянии формы нужно хранить порядок загрузки в виде массива id дисков.

Диски в разделе "Порядок загрузки" должны быть синхронизированы с дисками в разделе "Диски HDD".
Конкретно:

* Если диск удаляется из раздела "Диски HDD", то он должен пропадать и из раздела "Порядок загрузки".
* Если новый диск добавляется в раздел "Диски HDD", то он должен появится и в разделе "Порядок загрузки" в конце списка.
* Если у диска в разделе "Диски HDD" меняется название, то оно меняется и в разделе "Порядок загрузки". При этом id диска не меняется. id диска определяется при его добавлении и больше никогда не меняется.

## Постановка задачи. Требования

* Раздел нужно разместить в отдельной папке BootOrder в одной папке с формой. Назвать компонент BootOrder.
* Нужно расширить интерфейс CreateVmFormValues - добавить туда переменную состояния для хранения порядка загрузки (массив строк).
* Для новый переменной состояния задать начальное значение (пустой список).
* Для реализации drag-n-drop нужно подобрать подходящий npm пакет. Нужно аргументировать выбор пакета и согласовать выбор со мной. Я предлагаю использовать @dnd-kit/core и @dnd-kit/sortable пакеты.
